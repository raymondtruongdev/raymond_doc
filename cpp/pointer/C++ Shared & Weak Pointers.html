Shared & Weak Pointers

Giải pháp tối ưu khi nhiều đối tượng cần cùng sở hữu và theo dõi một tài nguyên duy nhất.

1

Shared pointer

Reference Counting (Đếm tham chiếu)
  • → Cơ chế: Mỗi khi copy một shared_ptr, số lượng chủ sở hữu Ref Count) tăng lên 1.
  • → Tự động hủy: Vùng nhớ chỉ bị giải phóng khi Ref Count trở về con số 0.
  • → An toàn: Tránh được Dangling Pointer vì tài nguyên luôn sống nếu còn ít nhất 1 pointer trỏ tới.
2

Circular Dependency

Lỗi này xảy ra khi Đối tượng A giữ shared_ptr tới B, và B cũng giữ shared_ptr ngược lại tới A. Kết quả: Ref Count không bao giờ về 0 -> Memory Leak vĩnh viễn!

3

Weak pointer

Người quan sát (Observer)

weak_ptr trỏ vào vùng nhớ của shared_ptr nhưng không làm tăng Ref Count.

Phá vỡ vòng lặp Circular Dependency hiệu quả.
Kiểm tra được dữ liệu còn tồn tại hay không qua expired(). 
// Cách dùng weak_ptr an toàn
void weakDemo() {
    auto sPtr = std::make_shared<int>(42);
    std::weak_ptr<int> wPtr = sPtr;

    // Chuyển sang shared để dùng (Locking)
    if (auto observed = wPtr.lock()) { 
        std::cout << *observed; // An toàn
    } else {
        std::cout << "Dữ liệu đã bị xóa!";
    }
}

Sử dụng loại pointer phù hợp:

Loại Pointer Mục đích chính Ghi chú
unique_ptr Độc quyền sở hữu Hiệu suất cao nhất, dùng cho 90% trường hợp cơ bản.
shared_ptr Chia sẻ quyền sở hữu Dùng khi một tài nguyên có nhiều "chủ" cùng quản lý.
weak_ptr Theo dõi tài nguyên Giải pháp triệt để cho Circular Reference.
