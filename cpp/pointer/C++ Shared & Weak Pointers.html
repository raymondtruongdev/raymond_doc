Shared & Weak Pointers

Khi nhiều đối tượng cùng sở hữu một tài nguyên

1. std::shared_ptr (Sở hữu chung)

Reference Counting (Đếm tham chiếu)

  • Cơ chế: Mỗi khi copy một shared_ptr, số lượng chủ sở hữu (Ref Count) tăng lên 1.
  • Tự động hủy: Vùng nhớ chỉ bị delete khi Ref Count trở về 0.
  • An toàn: Tránh được Dangling Pointer vì vùng nhớ luôn tồn tại chừng nào còn ít nhất 1 con trỏ trỏ tới.
// Sử dụng shared_ptr
void sharedDemo() {
    auto sPtr1 = std::make_shared<Data>(100); 
    {
        auto sPtr2 = sPtr1; // Ref Count = 2
        std::cout << sPtr1.use_count(); // 2
    } // sPtr2 hết scope, Ref Count = 1
    
} // sPtr1 hết scope, Ref Count = 0 -> Xóa Data

2. Thảm họa: Circular Dependency

Lỗi này xảy ra khi Đối tượng A giữ shared_ptr tới B, và B cũng giữ shared_ptr ngược lại tới A. Kết quả: Ref Count không bao giờ về 0 -> Memory Leak vĩnh viễn! 

struct B; // forward declaration
struct A { std::shared_ptr<B> bPtr; };
struct B { std::shared_ptr<A> aPtr; };

void leakDemo() {
    auto a = std::make_shared<A>();
    auto b = std::make_shared<B>();
    a->bPtr = b;
    b->aPtr = a; 
} // Hết hàm, a và b vẫn nắm giữ nhau -> Không bao giờ bị xóa!

3. Giải pháp: std::weak_ptr

Người quan sát (Observer)

weak_ptr trỏ vào vùng nhớ của shared_ptr nhưng không làm tăng Ref Count.

Phá vỡ vòng lặp Circular Dependency.
Kiểm tra được dữ liệu còn tồn tại hay không (expired()). 
// Cách dùng weak_ptr an toàn
void weakDemo() {
    auto sPtr = std::make_shared<int>(42);
    std::weak_ptr<int> wPtr = sPtr;

    if (auto observed = wPtr.lock()) { // Chuyển sang shared để dùng
        std::cout << *observed; 
    } else {
        std::cout << "Dữ liệu đã bị xóa!";
    }
}

Khi nào dùng con trỏ nào?

Loại Pointer Mục đích chính Ghi chú
unique_ptr Độc quyền sở hữu Nhanh nhất, nhẹ nhất, dùng 90% trường hợp.
shared_ptr Chia sẻ quyền sở hữu Dùng khi nhiều object cần truy cập cùng 1 resource.
weak_ptr Theo dõi tài nguyên Giải quyết Circular Reference, không giữ tài nguyên.
