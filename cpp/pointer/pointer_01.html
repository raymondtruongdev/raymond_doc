Basic Pointer in C++

Tìm hiểu về Raw Pointers, Smart Pointers, Lifecycle để kiểm soát bộ nhớ hiệu quả và tránh rò rỉ.

1. Raw Pointer

Vòng đời (Lifecycle)

  • • Khởi tạo: Dùng từ khóa new để cấp phát trên vùng nhớ Heap.
  • • Sử dụng: Truy cập giá trị qua toán tử giải tham chiếu * hoặc ->.
  • • Hủy bỏ: PHẢI dùng delete thủ công để giải phóng. 
// Ví dụ Raw Pointer nguy hiểm
void rawPointerDemo() {
    int* ptr = new int(10); 
    if (error()) return; // Memory Leak!
    delete ptr; 
}

3. Truyền Tham số & Quyền Sở hữu

⚠️ Chú ý: Khi bạn dùng std::move(uPtr), biến gốc sẽ bị rỗng (nullptr). Bạn KHÔNG THỂ sử dụng nó sau dòng lệnh đó.

Cách A: Truyền Tham chiếu (Borrowing)

Khuyên dùng

Đây là cách tốt nhất nếu bạn muốn hàm xử lý xong mà bạn vẫn nắm quyền sở hữu đối tượng.

// Hàm mượn (Borrowing)
void work(Entity& e) {
    e.doSomething();
}

void demo() {
    auto uPtr = std::make_unique<Entity>();
    work(*uPtr); // Truyền nội dung
    uPtr->stillHere(); // OK!
}

Tại sao tránh rò rỉ?

Vì hàm work không có quyền hủy. Quyền hủy bỏ vẫn nằm hoàn toàn ở uPtr trong phạm vi hàm gọi.

Cách B: Return Ownership (Lấy rồi trả lại)

Dùng khi hàm cần biến đổi chính cái pointer đó rồi bàn giao lại quyền quản lý cho bạn.

std::unique_ptr<Entity> transform(std::unique_ptr<Entity> ptr) {
    ptr->modify();
    return ptr; // Trả lại quyền sở hữu
}

void demo() {
    auto u = std::make_unique<Entity>();
    u = transform(std::move(u)); // Nhận lại
    u->stillHere(); // OK!
}

Cơ chế hoạt động:

Quyền sở hữu đi vào hàm qua move, sau đó đi ra khỏi hàm qua return. Đối tượng luôn có chủ sở hữu rõ ràng.

💡 Mẹo: Nếu logic đòi hỏi nhiều nơi cùng dùng một vùng nhớ lâu dài, hãy dùng std::shared_ptr. Khi đó bạn chỉ cần copy thay vì dùng std::move.
