Basic Pointer in C++

Tìm hiểu về Raw Pointers, Smart Pointers, Lifecycle để tránh rò rỉ bộ nhớ.

1. Con trỏ Thô (Raw Pointer)

Vòng đời (Lifecycle)

  • Khởi tạo: Dùng từ khóa new để cấp phát trên vùng nhớ Heap.
  • Sử dụng: Truy cập giá trị qua toán tử giải tham chiếu * hoặc ->.
  • Hủy bỏ: PHẢI dùng delete thủ công.
// Ví dụ Raw Pointer nguy hiểm
void rawPointerDemo() {
    int* ptr = new int(10); 
    if (error()) return; // Memory Leak!
    delete ptr; 
}

⚠️ Chú ý: Khi bạn dùng std::move(uPtr), biến uPtr đó sẽ bị rỗng (nullptr). Bạn KHÔNG THỂ dùng nó sau đó.

Cách A: Dùng tiếp sau hàm - Truyền Tham chiếu (Borrowing)

Đây là cách tốt nhất nếu bạn muốn hàm xử lý xong mà bạn vẫn nắm quyền sở hữu.

// Hàm nhận tham chiếu (không sở hữu)
void work(Entity& e) {
    e.doSomething();
}

void demo() {
    auto uPtr = std::make_unique<Entity>();
    work(*uPtr); // Truyền nội dung bên trong
    
    uPtr->stillHere(); // OK! uPtr vẫn còn dữ liệu
}

Tại sao tránh leak?

Vì hàm work không bao giờ gọi delete. Quyền hủy bỏ vẫn nằm hoàn toàn ở uPtr trong hàm demo.

Cách B: Hàm lấy rồi trả lại (Return Ownership)

Dùng khi hàm cần "biến đổi" chính cái pointer đó rồi đưa lại cho bạn.

std::unique_ptr<Entity> transform(std::unique_ptr<Entity> ptr) {
    ptr->modify();
    return ptr; // Trả lại quyền sở hữu
}

void demo() {
    auto uPtr = std::make_unique<Entity>();
    uPtr = transform(std::move(uPtr)); // Nhận lại
    
    uPtr->stillHere(); // OK! uPtr đã có dữ liệu mới
}

Cơ chế hoạt động:

Quyền sở hữu đi vào hàm qua move, sau đó đi ra khỏi hàm qua return. Bộ nhớ không bao giờ bị "vô chủ".

💡 Mẹo: Nếu logic của bạn thực sự đòi hỏi nhiều nơi cùng dùng một vùng nhớ lâu dài mà không biết ai sẽ là người cuối cùng, hãy cân nhắc std::shared_ptr. Khi đó bạn không cần std::move mà chỉ cần copy nó.
