Semantic HTML

Bài học chi tiết cho người mới học HTML

Giới thiệu Semantic Các thẻ Cấu trúc Sai lầm

1. HTML có các thẻ bọc khối để làm gì?

Khi mới học HTML, nhiều người thường thấy các thẻ như:

body, main, section, div, article, header, footer, aside, nav...

Các thẻ này chủ yếu dùng để:

  • Chia bố cục website thành từng khu vực rõ ràng
  • Giúp code dễ đọc và dễ bảo trì
  • Giúp Google hiểu nội dung trang web tốt hơn (SEO)
  • Giúp trình đọc màn hình hỗ trợ người khiếm thị
  • Giúp developer khác dễ hiểu cấu trúc dự án

Ví dụ dễ hiểu

Hãy tưởng tượng website giống như một ngôi nhà:

  • header → bảng tên trước nhà
  • nav → lối đi giữa các phòng
  • main → khu vực chính
  • section → từng phòng
  • article → một nội dung độc lập
  • aside → phần bên cạnh
  • footer → phần cuối trang

2. Semantic HTML là gì?

Semantic nghĩa là “mang ý nghĩa”.

Semantic HTML là cách dùng các thẻ HTML có ý nghĩa rõ ràng thay vì dùng toàn bộ là div.

❌ Không semantic

<div class="header"></div>
<div class="menu"></div>
<div class="content"></div>
<div class="footer"></div>

✅ Semantic HTML

<header></header>
<nav></nav>
<main></main>
<footer></footer>

Lợi ích lớn nhất

  • Code nhìn chuyên nghiệp hơn
  • Dễ scale project lớn
  • SEO tốt hơn
  • Dễ debug
  • Dễ làm việc nhóm

3. Vai trò từng thẻ Semantic HTML

body

Đây là phần chứa toàn bộ nội dung hiển thị trên trình duyệt. 

<body>
    Nội dung website nằm ở đây
</body>
Lưu ý: Mỗi trang HTML chỉ có duy nhất 1 thẻ body.

header

Dùng cho phần đầu trang hoặc đầu một khu vực.

  • Logo
  • Tên website
  • Menu
  • Search box
<header>
    <h1>My Website</h1>
</header>

nav

Chứa menu điều hướng. 

<nav>
    <a href="/">Home</a>
    <a href="/about">About</a>
</nav>
Không phải mọi link đều cần nav. Chỉ dùng nav cho nhóm link điều hướng chính.

main

Chứa nội dung chính của trang.

  • Mỗi trang chỉ nên có 1 main
  • Không đặt main bên trong footer/header/nav
  • Main là khu vực quan trọng nhất
<main>
    Nội dung chính
</main>

section

Chia nội dung thành từng phần theo chủ đề.

Ví dụ một website bán hàng có thể có:

  • Section sản phẩm nổi bật
  • Section đánh giá
  • Section tin tức
<section>
    <h2>Sản phẩm nổi bật</h2>
</section>
Section thường nên có heading như h2, h3.

article

Dùng cho nội dung độc lập, có thể tách riêng.

Ví dụ:

  • Bài blog
  • Post Facebook
  • Bài báo
  • Comment
<article>
    <h2>Học HTML cơ bản</h2>
    <p>Nội dung bài viết...</p>
</article>

aside

Nội dung phụ bên cạnh nội dung chính.

Ví dụ:

  • Sidebar
  • Quảng cáo
  • Related posts
  • Thông tin tác giả
<aside>
    Bài viết liên quan
</aside>

footer

Phần cuối trang web.

Thường chứa:

  • Copyright
  • Thông tin công ty
  • Liên hệ
  • Social links
<footer>
    Copyright 2026
</footer>

div

div là thẻ container chung chung, không có ý nghĩa semantic.

div chủ yếu dùng để:

  • Layout
  • Flexbox
  • Grid
  • CSS styling
  • Grouping elements
<div class="card">
    Nội dung card
</div>
Nguyên tắc quan trọng: Nếu có semantic tag phù hợp → ưu tiên semantic tag. Chỉ dùng div khi không có semantic phù hợp.

4. Cấu trúc HTML tối ưu cho website

Đây là cấu trúc semantic HTML phổ biến và tối ưu cho hầu hết website hiện đại. 

<body>

    <header>
        <nav>
            Menu
        </nav>
    </header>

    <main>

        <section>
            <h2>Hero Section</h2>
        </section>

        <section>
            <article>
                Bài viết 1
            </article>

            <article>
                Bài viết 2
            </article>
        </section>

        <aside>
            Sidebar
        </aside>

    </main>

    <footer>
        Footer
    </footer>

</body>

✅ Ưu điểm

  • Dễ đọc
  • Dễ SEO
  • Dễ mở rộng
  • Dễ maintain
  • Semantic rõ ràng

❌ Sai lầm phổ biến

  • Lạm dụng div
  • Không dùng main
  • Section không có heading
  • Nhồi nhiều semantic sai mục đích
  • Nested quá sâu

5. Khi nào dùng div thay vì semantic tag?

Tình huống Nên dùng
Menu điều hướng nav
Bài viết độc lập article
Chia khu vực theo chủ đề section
Container layout div
Sidebar aside
Nội dung chính main

6. Các lỗi người mới thường gặp

❌ Dùng div cho mọi thứ

<div class="header"></div>
<div class="footer"></div>

✅ Hãy dùng semantic tag

<header></header>
<footer></footer>

❌ Nested quá sâu

<div>
    <div>
        <div>
            <div>
            ...
            </div>
        </div>
    </div>
</div>

✅ Hãy giữ cấu trúc đơn giản

  • Code càng đơn giản càng tốt
  • Semantic rõ ràng
  • Ít div vô nghĩa
  • Dễ debug

7. Kết luận quan trọng

Hãy nhớ:

  • Semantic HTML giúp code chuyên nghiệp hơn
  • Không dùng div cho mọi thứ
  • Mỗi thẻ có ý nghĩa riêng
  • main chỉ nên có 1
  • section nên có heading
  • article là nội dung độc lập

Thứ tự tư duy tốt nhất:

  1. Xác định khu vực chức năng
  2. Chọn semantic tag phù hợp
  3. Chỉ dùng div cho layout
  4. Giữ cấu trúc rõ ràng
  5. Tránh nested sâu

Một frontend developer giỏi không phải là người dùng nhiều framework, mà là người tạo được cấu trúc HTML rõ ràng, semantic, dễ maintain và dễ scale.
