Làm chủ kỹ thuật Git Push

Khám phá cách đồng bộ mã nguồn, thiết lập upstream và quản lý lịch sử commit an toàn. Tài liệu này giúp bạn phân biệt rõ các tham số từ cơ bản đến nâng cao.

🚀 1. Git Push là gì?

Trong hệ thống Git, git push là hành động chuyển giao các commit từ local repository lên remote repository.

git push origin main

Lệnh đẩy nhánh "main" lên server "origin".

⚓ 2. Thiết lập Upstream (-u)

Recommended

git push -u origin main

Thiết lập mối liên kết bền vững. Git sẽ ghi nhớ nhánh remote cho các lần push/pull sau.

Alternative

git push origin main -u

Hoạt động tương tự nhưng không phổ biến trong các tài liệu chuẩn (Convention).

Tại sao cần Upstream?

  • ✓ Chỉ cần gõ git push (không cần tham số).
  • ✓ Tận dụng tính năng git status để biết local nhanh hay chậm hơn server.
  • ✓ Tự động đồng bộ khi thực hiện git pull.

⚠️ 3. Force Push & Quản trị rủi ro

HEAD là gì?

Là con trỏ vị trí hiện tại. git push origin HEAD giúp bạn đẩy nhanh nhánh đang làm việc mà không cần nhớ tên nhánh.

Lưu ý cực quan trọng:

Force push ghi đè lịch sử server. Chỉ dùng sau khi Rebase hoặc Reset nếu bạn thực sự hiểu mình đang làm gì.

So sánh mức độ rủi ro

Option Cơ chế Ứng dụng
--force Ghi đè tuyệt đối, bỏ qua kiểm tra remote. Nhánh cá nhân, sửa lỗi gấp.
--force-with-lease Chỉ ghi đè nếu remote không có commit mới. Làm việc nhóm (Teamwork).

🛠️ 4. Workflow tiêu chuẩn

1

Tạo nhánh & Làm việc

git checkout -b task-auth
2

Push lần đầu tiên (Setup Upstream)

git push -u origin task-auth
3

Làm việc nhóm & Force an toàn

git push --force-with-lease

Tổng kết nhanh

Cơ bản

git push origin [nhánh]

Tiện lợi

git push -u origin [nhánh]

Chuyên nghiệp

git push --force-with-lease

Nguy hiểm

git push --force
